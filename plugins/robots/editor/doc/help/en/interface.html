TRIK Studio user interface

 

Overview

TRIK Studio can work in two modes: in editing mode and in debugging mode. Editing mode allows to create robot control diagrams and to run them on a robot in generation mode. Debugging mode allows to run and debug programs on two-d model or on a real robot in interpretation mode.

Overview of an editing mode user interface is on a figure:

  1. Main menu contains basic operations and settings of TRIK Studio.
    • "File" menu:
      • New project — create a new project.
      • Recent projects... — open one of recently edited projects.
      • New DIagram — create new diagram in a current project.
      • Open... — open a saved project. Projects are saved on a disk with *.qrs extension.
      • Save — save current project.
      • Save as... — save current project into specified location. All subsequent saves will be done into selected file.
      • Save diagram as picture... — save current diagram as a picture on a disk.
      • Print — print a diagram or text using "Print" dialog from operating system.
      • Quit — exit program.
    • "Edit" menu:
      • Undo — undo last command.
      • Redo — redo a command that was undone.
      • Find... — open "Find" dialog that allows to search for block names, properties and so on.
    • "View" menu:
      • Zoom In — increase zoom level of a scene.
      • Zoom Out — decrease zoom level of a scene.
      • Show splashscreen — turn on/turn off showing of splashscreen during startup of TRIK Studio.
      • Panels — show or hide various dock windows.
      • To main page — shows main diagram of a project, even if it was closed accidentally.
    • "Tools" menu. Contents of this menu may depend on installed plugins and current system configuration.
      • Mouse gestures — open tab with available mouse gestures.
      • 2D model — switch TRIK Studio interpreter to 2D model mode. Robot program execution will be simulated on 2D model without the need of a real robot.
      • Real robot — switch TRIK Studio interpreter to real robot mode. Robot program will be executed on a computer and it will send commands to a real robot over WiFi, Bluetooth or USB, depending on used robot kit (TRIK, EV3 or NXT) and robot connection settings.
      • Robot settings... — open setting window for a robot, where current robot model, communication mode and various settings for currently selected robot model can be set.
      • Save as task... — save current diagram and 2D model as an exercise. Exercise is a special kind of save, where specified parts can not be edited (like 2D model scene or sensor configuration). When you select this menu item, a dialog where you can select fixed parts will appear. For more information see Exercises.
      • Upload runtime — upload runtime to a robot. Runtime allows to execute programs on a real robot autonomously. This action needs to be done only once for each robot, before you execute your very first program on it. Some robots, like TRIK, come with pre-installed runtime, so this action may not be needed at all. Some modes, like execution of a program on NXT/EV3 in interpretation mode, will work on default runtime, in such cases this action is also not needed. General rule is that you shall not use this menu item until you tried to execute a program on a real robot and it did not worked. For more info see corresponding help topics for concrete robot kit.
      • Generate code — open text editor tab with a code generated by current diagram on a selected textual language. This item depends on a selected textual language and may look like "Generate TRIK code ", "Generate pascalABC code", "Generate FSharp code" and so on. Code will not be uploaded to a robot or executed, but it can be edited and executed later.
      • Upload program — generate code by current diagram and upload it on a robot, but don't run it.
      • Run program — generate codeby current diagram, upload it on a robot and execute it there.
      • Stop robot — abort execution of a current program and stop robot motors.
      • Connect to robot — connect to a robot. Robot shall be turned on. This item is only available when selected program execution mode requires constant connection to a robot (for example, interpretation over Bluetooth or WiFi).
      • Run — execute current program (diagram or textual code) on 2D model or a robot in interpretation mode. Available only when interpretation mode or 2D mode is selected.
    • "Settings" menu (for more information about setings see Settings dialog):
      • Settings... — open settings window.
      • Show grid — turn on/turn off grid on a scene.
      • Show alignment — turn on/turn off vertical and horisontal alignment lines. Alignment lines simplify automatical alignment of blocks relative to each other.
      • Switch on grid — turn on/turn off snapping blocks to grid.
      • Switch on alignment — turn on/turn off snap to alignment lines.
      • Show all text — turn on/turn off showing of all block properties on a scene. When off, only most important properties are shown and other properties can be viewed and edited in property editor. When on, all properties are shown and can be edited directly on scene.
    • "Help" menu :
      • Small help — open TRIK Studio help system.
      • About... — see quick information about TRIK Studio (version, site).
      • About Qt... — see information about Qt library that was used to create TRIK Studio.
      • Check for updates — get nformation about new version of TRIK Studio or one of installed plugins from the internet. It also launches autoupdate if needed.
  2. "File" toolbar repeats most commonly used actions available from "File" menu.
  3. "Edit" toolbar contains Undo and Redo buttons.
  4. "View" toolbar has buttons that allows to scale the scene. Scene can also be scaled by mouse wheel when mouse cursor is over a diagram.
  5. "Interpreter" toolbar contains buttons that allow to run or stop the execution of a program, buttons for changing interpretation modes between 2D model or real robot, and a button that opens "Settings" window (this window is also available from "Tools" menu). When working with real robot this toolbar also has button that allows to connect to robot.
  6. Editor window shows current diagram and allows to edit it. When diagram is interpreted (on 2D model or on a real robot), currently active block is highlighted here.
  7. Property editor allows to see and edit properties of the element selected in editor.
  8. Configure devices allows to configure which sensors to which ports are connected. It can also be configured from "Settings" window.
  9. Palette contains available blocks which can be added into diagram. Each block corresponds to robot command (such as "Motors forward" or "Wait for touch sensor") or algorithmic construct, like "If" or "Start parallel task". Palette contents depend on selected robotic kit and program execution mode (real robot or 2D model), grey blocks are available for current kit, but disabled for current mode (for example, robot communication blocks are disabled in 2D model). Blocks from other kits will not even be shown on a palette. More info on available blocks and a process of diagram creation see in "Programming".
  10. Watch List shows all variables used in program with their values during the execution, including sensor variables (special read-only variables that contain current sensor readings). When program is executed on 2D model or on real robot in interpretation mode, sensor variables are updated with values from robot in a real time (once in about 200 ms), in generation mode they are not used (and not updated, so don't worry if all sensor readings are zero).
  11. Status bar shows current TRIK Studio mode (editing or debugging) and allows to quickly switch between the modes. Just click on a status bar to change mode.
  12. "Modes" toolbar also shows current TRIK Studio mode and also allows to switch between modes, and can be more convenient.

Debugging interface of TRIK Studio is on the picture below:

  1. Main menu, toolbars and status bar are the same as in the editing mode.
  2. "2D model" window opens if 2D model was selected as execution mode. This window allows to specify enwironment in which robot will work, specify robot sensors and execute a program seeing how simulated robot behaves when program is running. For more information about 2D model see "2D model window". When program is interpreted on a real robot over WiFi, Bluetooth or USB this window stays closed, and a diagram with running program is shown instead.
  3. "Watch List" window, like in editing mode, shos sensor variables and all variables that were defined in a program. Note that variables defined in program will be shown only when program is executed and interpreter gets to their definition.
  4. "Sensors state" window shows readings of the sensors as a graph, and is updated in real time when program is executed in interpretation mode.

TRIK Studio settings dialog

Settings dialog is opened by selecting "Settings" — "Preferences..." menu item or pressing "Robot settings" button on a toolbar. This window contains two panels — left, that shows a list of settings tabs, and right, that shows details about current settings tab and allows to edit various setting on selected tab. "Import" and "Export" buttons in the bottom of the window allow to save current settings as a file and to load this file on another computer, thus copying or restoring settings.

"Behaviour" tab

  • Language — user language of a system. TRIK Studio shall be restarted for change in this setting to take effect. "<System language>" setting means that TRIK Studio language will be determined by current language of operating system. All language settings can be overriden with "--no-locale" command-line parameter when starting TRIK Studio.
  • Autosave — turns on or off autosave, also it is possible to set autosave intervals (in seconds). Note that autosave can be quite slow, so do not set it to be too often.
  • Gestures — turns on or off mouse gesture recognition.
  • Delay after gesture — how long does sytem wait after mouse gesture before it attempts to recognize it. Delay allows to draw a gesture with multiple mouse strokes. Delay value is in milliseconds.
  • Check for updates on start — turn on or of checking for updates on startup. Requires internet connection.
  • Touch mode — turn on or off optimization of the user interface for touch screens.

"Miscellaneous" tab

  • Graphics/Antialiasing — turns on or off drawing with antialiasing, improves looks of a diagram, but slightly decreases performance.
  • Other/Show splashscreen — turn on or off showing of splashscreen with TRIK Studio logo on startup. Turning it off does not make startup faster.
  • Other/Limit recent projects list — how many projects shall be shown on "Recent projects" in "File" menu.
  • Images — what icon set to use for diagrams. Icon set is purely aestetic.
  • Toolbars/Size — set buttons size on various toolbars.

"Editor" tab

  • Font/Use some of system fonts — use one of the fonts installed on an operating system as a font for labels on a diagram. Note that label positions on a block are fixed, so changing this setting may result in labels overlapping each other. But if you don't like default font that is bundled with TRIK Studio, you are free to experiment, as font is purely aestetic and all font changes are reversible.
  • Grid/Show grid — turn on or off drawing of alignment grid on the editor window. Equivalent to corresponding "Settings" menu item.
  • Grid/Activate grid —turn on or off snapping of blocks to grid.
  • Grid/Show alignment — turn on or off drawing of alignment lines on the editor window. Alignment lines are shown when block is on one horisontal or vertical line with other block and help to align blocks.
  • Grid/Activate alignment — turn on or off snapping blocks to grid lines.
  • Grid/Width — width of grid lines. It may be helpful if a monitor or projector does not show grid lines brightly.
  • Grid/Cell size — size of one grid cell in pixels. Default size is such that a block covers exactly four cells.
  • Node Elements/Drag area — size of an area which can be used to resize block. Note that the only resizable block in TRIK Studio is "Comment.
  • Edge/Line mode — drawing mode of connectors between blocks.
    • broken — lines are drawn as polylines with turning points that can be added by user (this is default setting).
    • square — lines are drawn as polylines with each segment of a line is parallel to the coordinate axes. Turning points are added automatically in this case.
    • curved — lines are drawn as curves with curvature that can be set by the user.
  • Edge/Loop edges indent — sets an indent from a block of a line that starts and end in the same block.
  • Embedded linkers/Size — a size of the embedded linker, i.e. a circle near a block which allows to create an edge between the blocks (see Creating links for more info).
  • Embedded linkers/Indent — a distance between the linker circle and the block.
  • Labels/Enable move — allow or disallow to move text labels of the blocks on a diagram.
  • Labels/Enable resize — разрешить/запретить менять размер области отображения для надписи.
  • Palette/Representation — выбор между режимом отображения иконок и названий или только иконок в палитре.
  • Count of items in a row — количество иконок на строку палитры при выбранном режиме "Иконки".

"Robots" tab

Состав этой вкладки зависит от установленных подключаемых модулей поддержки конструкторов и от текущего выбранного конструктора. Информацию о настройках для конкретного конструктора можно посмотреть в разделе документации, ему соответствующему. Типовые элементы этого окна таковы:

  • Конструктор — конструктор, для которого пишутся программы в среде. От этой опции зависит список доступных блоков в палитре и доступных действий на панели инструментов.
  • Тип модели робота — выбор типа модели исполнения для выбранного конструктора. В большинстве случаев это двумерная модель или реальный робот, но для некоторых конструкторов одна из моделей может быть недоступной, в этом случае этот пункт в окне не отображается.
  • Настройки соединения — специфичные для конструктора настройки подключения к роботу, такие как IP-адрес робота для ТРИК или COM-порт Bluetooth-соединения для Lego NXT. Если соединение в выбранном режиме не требуется (как на рисунке), не отображается.
  • Настройки сторонних программ — специфичные для конструктора настройки программ, не поставляемых в комплекте с TRIK Studio, которые должны быть установлены на компьютере. Для большинства режимов работы дополнительные установленные программы не нужны, так что соответствующие пункты не отображаются в окне.
  • Настройка сенсоров — здесь указывается, какие сенсоры подключены к портам робота.
  • Запуск после загрузки — запускать программу сразу после загрузки её на робот или нет. Применимо только для некоторых режимов некоторых конструкторов (например, для Lego NXT в режиме генерации).
  • Шум/Реалистичные сенсоры — включить/выключить эмуляцию неточностей считывания показаний с сенсоров в 2D модели.
  • Шум/Реалистичные моторы — включить/выключить эмуляцию неточностей работы моторов в 2D модели.
  • Шум/Уровень приближения — насколько точно выполняется эмуляция погрешностей.
  • Интервалы обновления графиков/Сенсоры — интервал в миллисекундах, когда добавляется новая точка с показанием сенсора.
  • Интервалы обновления графиков/Масштабирование — интервал в миллисекундах, когда график автоматически изменяет свой масштаб, чтобы оптимально отобразить значения.
  • Интервалы обновления графиков/Текстовая информация — интервал в миллисекундах, когда на графике обновляются надписи со значениями.

"Shortcuts" tab

Позволяет задать или изменить горячие клавиши для наиболее часто используемых действий. Для этого надо выбрать ячейку, соответствующую действию, и в строке "Сочетание" внизу окна нажать нужное сочетание клавиш. Кнопка "Очистить" удаляет сочетание клавиш из ячейки.

Окно двумерной модели робота

Режим работы с двумерной моделью полезен для отладки программ, а также в случае отсутствия реального робота. В этом режиме программы выполняются в пошаговом режиме, но в этом случае команды не передаются роботу, а исполняются для виртуальной модели робота, отображающейся на экране. Осуществляется моделирование трехколесной тележки. Считается, что изначально робот находится в бесконечном пустом пространстве, для пользователя существует возможность расставлять препятствия в виде стен, рисовать цветные линии и области на полу. При этом эмулируется работа большинства реальных датчиков конструктора: например, датчика касания, ультразвукового датчика расстояния, датчиков цвета и света для NXT, ИК- и УЗ-датчиков расстояния и датчика света для ТРИК. Двумерная модель может иметь устройства, недоступные на реальном роботе, например, возможность рисовать цветные линии маркером на полу (как "черепашка" LOGO). Внешний вид и набор устройств двумерной модели зависит от выбранного конструктора.

Окно двумерной модели робота TRIK Studio доступно для отображения, если в качестве типа модели робота в диалоге настроек робота или на панели инструментов выбран пункт "Двумерная модель" (см. раздел Диалог настроек TRIK Studio). В таком случае при переключении в режим отладки будет открыто следующее окно:

Данное окно также открывается автоматически, если в этом режиме начать выполнение программы с помощью пункта меню или панели TRIK Studio.

  1. Сцена двумерной модели. Представляет собой виртуальное пространство, в котором будет двигаться модель робота, обозначенная соответствующей иконкой.
  2. Панель редактирования модели мира содержит следующие кнопки:
    • Курсор — переключиться в режим выбора и перестать рисовать стены/линии и т.д. Горячая клавиша: N.
    • Стена — добавить на сцену препятствие в виде стены. Препятствие отображается в виде "кирпичной" линии, при касании такой линии сенсором касания или при попадании в зону действия сенсора расстояния происходит срабатывание сенсора, робот не может проехать сквозь стену. При добавлении с нажатой клавишей Shift стена помещается либо под прямым углом, либо под углом в 45 градусов относительно границ окна. Горячая клавиша: W.
    • Линия — добавить на сцену прямую линию. При этом срабатывают находящиеся над линией сенсоры света и соответствующие сенсоры цвета. При добавлении с нажатой клавишей Shift линия помещается либо под прямым углом, либо под углом в 45 градусов относительно границ окна. Горячая клавиша: L.
    • Эллипс — добавить на сцену эллипс. При этом срабатывают находящиеся над эллипсом сенсоры света и соответствующие сенсоры цвета. При добавлении с нажатой клавишей Shift добавляется окружность. Горячая клавиша: E.
    • Карандаш — добавить на сцену нарисованную мышью линию. При этом срабатывают находящиеся над линией сенсоры света и соответствующие сенсоры цвета. Горячая клавиша: P.
  3. Эмулятор экрана и кнопок робота отображает вывод программы на экран робота и позволяет эмулировать нажатие на его кнопки (кликом мыши по соответствующей кнопке на роботе).
  4. Панель управления портами позволяет менять конфигурацию и расположение сенсоров модели робота. Для каждого порта можно выбрать интересующий датчик, и он автоматически добавится на сцену. При изменении конфигурации сенсоров из настроек сенсоры в двумерной модели автоматически подстраиваются. При этом будет считаться, что сенсор соединен с роботом, то есть при движении иконки робота сенсоры будут двигаться вместе с ним, даже если они находятся на некотором расстоянии от модели робота. Рядом с ультразвуковым или инфракрасным датчиком отображается серый конус, обозначающий область, в которой этот датчик будет регистрировать препятствия.
  5. Панель управлениямоторами позволяет задать, какой из портов моторов будет соответствовать левому и правому колесу. Под ней расположена панель управления физикой, которая позволяет настраивать следующие параметры:
    • Реалистичная физика более точно эмулирует силы, действующие на робот: он поворачивает при столкновении со стеной, разгоняется с ускорением и т.д. В таком режиме гораздо сложнее предсказать поведение робота, поэтому по умолчанию он отключён.
    • Реалистичные сенсоры — включить/выключить эмуляцию неточной работы сенсоров. То же можно сделать через окно настроек.
    • Реалистичные моторы — включить/выключить эмуляцию неточной работы моторов. То же можно сделать через окно настроек.
  6. Сетка позволяет включить отображение на сцене сетки и привязки стен к её узлам. Слайдер рядом позволяет настроить размер сетки.
  7. Панель управления скоростью позволяет задать скорость симуляции. При выборе низкой скорости одной секунде модельного времени соответствует несколько секунд реального, при выборе высокой скорости — наоборот. Здесь же отображается текущее модельное время, прошедшее с запуска программы на исполнение. Может отличаться от реального времени (зависит от настроек скорости и возможностей компьютера).
  8. Линейка показывает расстояния на сцене, в сантиметрах (относительно размера робота).
  9. Кнопка возврата на старт устанавливает робот в исходное положение, задаваемое красным крестиком на сцене. Крестик можно вращать вокруг своей оси, выделив его и таская конец появившейся стрелки, этим задаётся начальное направление робота.
  10. Кнопка "Старт/стоп" Позволяет запустить или остановить программу.

Окно двумерной модели можно отделить от остального интерфейса TRIK Studio, потащив его за заголовок. Вернуть окно на место можно, нажав на кнопку закрытия окна (окно не закроется, а вернётся в главное окно TRIK Studio).

Некоторые важные функции двумерной модели доступны при нажатии на правую кнопку мыши на сцене:

  1. Режим таскания сцены позволяет перемещать сцену с роботом, стенами и линиями при клике и перетаскивании на свободный участок сцены.
  2. Режим выделения позволяет выделять группу стен, линий и других частей модели при клике и перетаскивании на свободный участок сцены.
  3. Сохранить модель мира позволяет сохранить нарисованную сцену в виде XML-файла, который потом может быть загружен и использован в другом проекте.
  4. Загрузить модель мира позволяет загрузить ранее сохранённую модель мира.
  5. Очистить всё удаляет все стены и цветные линии. Робот при этом остаётся на сцене.
  6. Очистить пол от следов робота удаляет все цветные линии, нарисованные маркером робота. Остальные элементы остаются на сцене.

При выделении линии, карандаша или эллипса появляется всплывающее окно, позволяющее настраивать их цвет и толщину:

При выделении робота появляется следующее всплывающее окно:

  1. Центрирование на роботе включает или выключает центрирование камеры на роботе. При включённом режиме центрирования сцена автоматически перемещается, выставляя робот в центр, если он выезжает за границы видимого участка.
  2. Кнопка возврата на старт работает так же, как кнопка в основном окне двумерной модели — возвращает робот на исходную позицию, обозначаемую красным крестиком.
  3. Толщина маркера позволяет задать толщину следа, оставляемого роботом при рисовании маркером.
